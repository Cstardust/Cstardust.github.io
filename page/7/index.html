5. 关于（几乎就是不可以）返回局部变量的引用

  • 函数返回local variable引用（无论是否是常量引用）是不可以的。(条件是函数调用处并没有用常量引用来接收)
    • 理解：编译器根据 local 变量造了一个const局部临时量 让const引用绑定在这个临时变量上，可这个变量还是一个局部变量。函数结束这个局部的临时量时该销毁还是销毁了。
  • 对于函数中的一个变量，如果不是局部变量，而是函数传参传入的一个引用，那么返回时当然可以返回该变量的引用（无论是否常），因为这个变量在函数结束时并不会被销毁。

7. 静态成员函数指针 非静态成员函数指针

  • 问题： 在使用sort()函数时，第三个函数只能是 全局函数指针或者静态成员函数指针，不能是非静态成员函数指针。
  • 解释： 非静态成员函数的参数中有一个隐式绑定的this指针(因为非静态成员函数需要由具体对象来调用)，而sort第三个参数需要得是一个接受两个参数的函数指针，所以不可。如果是静态成员函数的话，sort时直接使用cmp即可，无需依赖于实际对象。

关于默认参数

  • 默认参数只能指定一次。在函数的声明和定义中只能在一处指定参数的默认值。如果两处都指定为非法；
  • 编译器把这种情况作为非法处理的原因是：在声明和定义中如果指定了不同的默认值，编译器将无法判断采用哪个值作为参数的默认值。

EPOLLRDHUP

这两个事件其实是一个东西，分别对应poll和epoll，通常用来判断对端是否关闭，但是当你对某个socket注册POLLIN和POLLRDHUP(EPOLLIN和EPOLLRDHUP)时，在对端关闭时，对于poll来说会一直触发POLLIN + POLLRDHUP事件，epoll也会触发EPOLLIN + EPOLLRDHUP事件，是否一直触发要看epoll是工作在LT模式下还是ET模式下。

所以，当我们使用POLLRDHUP（EPOLLRDHUP）事件来判断对端是否关闭时，POLLRDHUP（EPOLLRDHUP）事件的处理应放在POLLIN（EPOLLIN）事件的前面，避免将对端关闭当做一次读事件，而无法处理POLLRDHUP（EPOLLRDHUP）事件

常量左值引用可以绑定在非常量，字面值等

  • 引用的类型必须和其所引用的类型一致

  • 例外：const引用可以绑定非常量的对象，字面值，甚至是一般表达式 const int &r = 88

  • 为什么可以这样呢？

    • 首先，我们需要知道当常量引用被绑定到另一种类型上时，发生了什么

      在这种情况下，r绑定了一个临时量对象
      临时量对象：当编译器需要一个空间来暂存表达式的求值结果时临时创建的一个未命名对象。

    • 若r不是const，则执行了上述初始化过程之后。因为r不是常量，就意味着允许对r进行赋值，这样就会改变r所引用的对象的值。但是！在这时！r所引用的变量是临时量对象temp，而非最开始的变量v；而我们改变r的值是意图改变v的值，故行为目的与预期不符。

    • 所以c++就把将非const引用变量绑定到非常量对象，字面值，不同类型的对象行为定义为非法；而const引用则可以绑定到非常量对象，字面值，不同类型对象上。

关于const&refernce使用

  • 引用（const引用） 一般用于函数传参 。
  • 引用的底层就是指针
  • 关于const成员函数：
    • 不会改变数据内容的成员函数，马上加上const。
    • 原因：常对象只能调用常函数，非常对象可以调用常函数和非常函数
      • 所以像一些print(),show(),getXxx()...这些不会改变数据内容的函数，应加上const，不然常对象无法调用他们
    • const成员函数和非const成员函数构成重载
  • 传参：pass by value is be to replaced by pass by reference(to const)(当然如果碰到需要传字符（就一个字节)的话，也可以传字符（指针四个字节））
  • （const）reference 作为函数参数比（const）pointer好在哪里？

enum class

  • 枚举类型使我们可以将一组整型常量组织在一起，和类一样，每个枚举类型定义了一种新的类型。枚举属于字面值常量类型。

  • 枚举类型

    • **限定作用域(enum class / enum struct)**：c++11引入
      • enum class open_modes{input , output , append};
    • 不限定作用域(enum)
      • enum color{red , yellow , green}

  • 限定作用域的枚举类型中，枚举成员的名字遵循常规的作用域准则，并且在枚据类型的作用域外不可访问枚举成员的名字。

  • 不限定作用域的枚举类型中，枚举成员的作用域与枚举类型本身的作用域相同。

  • 如果我们没有显示的提供初始值，则当前枚举成员的值等于之前的枚举成员值+1

  • 指定enum的底层类型

    • 默认情况下限定作用域的enum成员类型为int
    • enum class value:double{...}
    • 不限定作用域的不存在默认类型。

  • 限定作用域的枚举类型不可以隐式转换

    • 只能使用static_cast<>()
    • int val = static<int>(A::enum_int_1);
    • 传统不限定作用域的枚举类型会发生隐式转换

inl

  • 在内联函数较多的情况下，为了避免头文件过长、版面混乱，可以将所有的内联函数定义移到一个单独的文件中去，然后再 用#include指令将它包含到类声明的后面（类的头文件的底部）。这样的文件称为一个内联函数定义文件。

  • inl文件中也可以包含头文件的，因为内联函数中可能包含其他文件中定义的东西。

  • 按照惯例， 应该将这个文件命名为filename.inl，其中filename与相应的头文件和实现文件相同。

    • 作用域：本.c文件中，不会被其他文件看见。在其他.c文件中可以命名同名的函数。

  • 非static修饰的全局函数：

    • 作用域：整个工程。其他.c文件中不能有同名的。不然链接时会redefition。

static函数出现在类中

  • 表示这个函数属于这个类而非某一对象，可通过类名直接访问

  • 在该static函数中不可以访问类内非静态成员（变量和函数）。

    • 因为在类加载内存时就会给类内static成员分配内存
    • 而在产生对象时才会给非静态成员分配内存。
