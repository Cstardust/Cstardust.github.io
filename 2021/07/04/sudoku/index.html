全局变量 静态static全局变量 const全局变量 静态变量

  • 参考 不错的博客
  • 一个挺基础的问题。上学期大一的时候遇见过没整明白。。想想可能是VS编译时做了某些优化吧。。现在直接vscode中用命令行编译条理清晰了不少。
  • 今后应该可以告别这个问题了。

    （非const非static)全局变量：全局作用域 (跨文件) 生命周期=程序生命周期

    • 即在所有文件中均可见。在整个源程序中只可以定义一个这个名字的全局变量。不同文件间同名的全局变量会发生冲突。
    • 全局/静态存储区：全局变量和静态变量被分配到同一块内存
    • 反面示例 error: multiple definition

  • 正解：在一个源文件中定义，在其余需要使该变量到的文件中使用extern声明。

  • 上述string换成int也一样。

静态变量：生命周期=程序生命周期

  • 作用域与定义在哪里有关。

静态static全局变量：文件作用域 生命周期=程序生命周期

  • 全局静态变量的作用域只在本文件内有效。
    • 即：即便两个文件中定义了同名的全局静态变量，也不会发生冲突。他们是两个变量。

const全局变量：文件作用域 生命周期=程序生命周期

  • 经我实验应该是文件作用域
  • 即 作用域 , 生命周期 方面等同static全局变量

enum class

  • 枚举类型使我们可以将一组整型常量组织在一起，和类一样，每个枚举类型定义了一种新的类型。枚举属于字面值常量类型。

  • 枚举类型

    • **限定作用域(enum class / enum struct)**：c++11引入
      • enum class open_modes{input , output , append};
    • 不限定作用域(enum)
      • enum color{red , yellow , green}

  • 限定作用域的枚举类型中，枚举成员的名字遵循常规的作用域准则，并且在枚据类型的作用域外不可访问枚举成员的名字。

  • 不限定作用域的枚举类型中，枚举成员的作用域与枚举类型本身的作用域相同。

  • 如果我们没有显示的提供初始值，则当前枚举成员的值等于之前的枚举成员值+1

  • 指定enum的底层类型

    • 默认情况下限定作用域的enum成员类型为int
    • enum class value:double{...}
    • 不限定作用域的不存在默认类型。

  • 限定作用域的枚举类型不可以隐式转换

    • 只能使用static_cast<>()
    • int val = static<int>(A::enum_int_1);
    • 传统不限定作用域的枚举类型会发生隐式转换

检验文件是否存在： access函数

  • 头文件：unistd.h
  • 功 能: 确定文件或文件夹的访问权限。即，检查某个文件的存取方式，比如说是只读方式、只写方式等。如果指定的存取方式有效，则函数返回0，否则函数返回-1。
  • 用 法: int access(const char *filenpath, int mode); 或者int _access( const char *path, int mode );
  • mode
    • 00 只存在, 02 写权限, 04 读权限, 06 读和写权限

inl

  • 在内联函数较多的情况下，为了避免头文件过长、版面混乱，可以将所有的内联函数定义移到一个单独的文件中去，然后再 用#include指令将它包含到类声明的后面（类的头文件的底部）。这样的文件称为一个内联函数定义文件。

  • inl文件中也可以包含头文件的，因为内联函数中可能包含其他文件中定义的东西。

  • 按照惯例， 应该将这个文件命名为filename.inl，其中filename与相应的头文件和实现文件相同。

静态全局函数和全局函数

  • static修饰的全局函数

    • 作用域：本.c文件中，不会被其他文件看见。在其他.c文件中可以命名同名的函数。

  • 非static修饰的全局函数：

    • 作用域：整个工程。其他.c文件中不能有同名的。不然链接时会redefition。

static函数出现在类中

  • 表示这个函数属于这个类而非某一对象，可通过类名直接访问

  • 在该static函数中不可以访问类内非静态成员（变量和函数）。

    • 因为在类加载内存时就会给类内static成员分配内存
    • 而在产生对象时才会给非静态成员分配内存。
