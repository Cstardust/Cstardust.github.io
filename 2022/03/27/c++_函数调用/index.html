内存地址与内存空间关系

寻址方式

  • 按字节寻址，那么一个地址背后就代表8个bit的内存单元；
  • 按字（16位）寻址，那么一个地址背后就代表16bit的内存单元。
  • 按字（32位）寻址，那么一个地址背后就代表32bit的内存单元
  • 所以对于同一存储器，按照不同的寻址方式，会有不同的寻址范围，也就是会有不同的寻址地址，但是总容量不变。

按字节寻址

  • 下面举例按字节寻址方式
  • 一个地址背后是8bit（1Byte）的内存单元
  • 内存地址只是一个编号，代表一个内存空间。那么这个空间是多大呢？
    • 计算机中存储器的容量是以字节为基本单位的。也就是说一个内存地址背后是一个字节（8bit）的存储空间。
  • 例如经常说32位的操作系统最多支持4GB的内存空间，也就是说CPU只能寻址2的32次方（4GB）。这里默认的寻址方式是按字节寻址。注意这里的4GB是以Byte为单位的，不是bit。
    • 4G=4*1024M(Byte)=4*1024*1024Kb(Byte)，也即按字节寻址会有2^12个地址，而每个地址背后会有8个bit的空间，故总空间大小为2^15bit
  • 所以说用4位16进制表示的内存地址和用8位16进制表示的内存地址，其实都是代表一个8bit的存储空间而已：


汇编复习(x86)

.text和.data .rodata .bss 和 stack

  • 指令（代码）都在.text段中
  • .data .rodata中是数据。（全局变量，地址啥的）

ss sp bp（指向栈stack的）

  • ss：存放栈stack的段地址（配合sp、bp使用）

sp esp

  • sp：stack pointer：栈指针寄存器（16位CPU）
    • 其内存放着一个指针，该指针永远指向系统栈最上面（地址最高的）一个栈帧的栈顶
    • esp：extended stack pointer：32位cpu
    • rsp：64位cpu
    • sp,esp,rsp都是指向栈顶的寄存器
  • esp中的指针指向的内存空间有内容，该指针指向的是栈顶地址最低的归本栈帧（有内容）的内存单元

bp ebp

  • ebp中的指针（地址）所指向（对应）的内存空间不归本栈帧（里没有内容）。指针指向的是栈底有内容的下一内存单元。

  • bp：base pointer：基数指针寄存器

    • 其内存放着一个指针，该指针永远指向系统栈最上面（地址最高的）一个栈帧的底部
    • ebp：extended base pointer
    • rbp
    • bp ebp rbp都是指向栈底的寄存器

  • SP、ESP就是一直指向栈顶的指针,而EBP只是存取某时刻的栈顶指针,以方便对栈的操作,如获取函数参数、局部变量

  • 和堆栈指针SP联合使用，作为SP校准使用的，只有在寻找堆栈里的数据和使用个别的寻址方式时候才能用到

    • 比如说，堆栈中压入了很多数据或者地址，你肯定想通过SP来访问这些数据或者地址，但SP是要指向栈顶的，是不能随便乱改的，这时候你就需要使用BP，把SP的值传递给BP，通过BP来寻找堆栈里数据或者地址．一般除了保存数据外,可以作为指针寄存器用于存储器寻址,此时它默认搭配的段寄存器是SS-堆栈段寄存器.BP是16位的,再扩充16位就是EBP,用于32位编程环境的.一般高级语言的参数传递等等,转换为汇编后经常由BP/EBP来负责寻址\处理.

    • SP,BP一般与段寄存器SS 联用，以确定堆栈寄存器中某一单元的地址，SP用以指示栈顶的偏移地址，而BP可 作为堆栈区中的一个基地址，用以确定在堆栈中的操作数地址。

  • 也即，栈中有东西的内存单元对应的地址是[esp,ebp)
  • 也即是，一个栈帧的有效地址是[esp,ebp)
    • [esp,ebp)的地址对应的内存单元的的内容，都是本函数中的局部变量等，而ss[ebp往高的，可能是另一个函数的栈帧，也有可能是进入本函数栈帧前保留的信息（有关上一个函数）。在离开本函数时，需要通过pop ebp来弹出恢复。（进入函数时push ebp，离开函数时pop ebp）

push和pop（栈自高向低使用）。

  • 栈顶：低地址
  • 栈底：高地址
  • push：从栈顶push
    • esp向低移动，再向移出来的内存中放入内容。esp-4,再放int型内容
  • pop：从栈底pop
    • esp指向的内容弹出，esp向高移动

CS IP （指向.text段（代码段）的）

  • CS为代码段寄存器，IP为指令指针寄存器，它们指示了CPU当前要读取指令的地址
  • 他们指向的指令位于.text段

函数调用过程

  • 调用函数：
    1. 参数压栈（传入的参数）
    2. call 函数名 a.保存下一指令地址 b.跳转到目标函数
    3. 进入函数 { ：
      保留现函数栈底 ，
      开辟栈帧 ，
    4. 函数代码段
    5. 离开函数 } ：
      栈帧回退 ，
      取回函数调用方的函数栈底，
      跳转到函数调用处的下一条指令（ret，对应sum）
