对象的优化三原则

实际上
以下都不考虑编译器自动做出的优化

总

  • 传参数传递引用（可以减少一或多个构造和析构函数）
  • 函数返回对象的时候，有限返回临时对象而非定义好的局部对象
  • 接收返回值是对象的函数调用时，尽量在定义时初始化接收、而非定义好了再用函数返回值去赋值

例子

初：不传引用、不返回临时对象

  • 耗费：
    • 形参会调用拷贝构造函数
    • 返回对象时，会在main栈帧调用拷贝构造函数，以函数内的对象为蓝本构造出一个临时对象。然后这个main栈帧上的临时对象再拷贝赋值给main调用处的，之前已经定义好的t。
    • 还要对函数中的局部对象tmp进行析构

传引用

  • 形参用引用，节省掉对形参的拷贝构造函

返回临时对象

  • 返回构造的临时对象，而非定义好的有地址的对象。
  • 省去了对函数内局部对象的构造和析构。直接在main栈帧上调用拷贝构造和析构。也就是少了一对构造和析构。

std::move 和 std::forward

move 移动语义

  • move：编译器强制将一个对象（无论左右）认为是一个右值。
    • 目的：为了匹配到想让其构造的函数（因为右值变量本身是一个左值，这会使得其去调用左值引用匹配的函数，与我们预期不符）
  • move本身不移动内存里的内容，只是让编译器识别他是个右值，并交付给相应函数
  • 虽然move本身不移动，但是正常来讲，move使得这个传入的参数匹配到了处理右值的函数，那么，那个函数正常来讲就会把传入的这个对象所管理的内存交付给别人，使得传入的对象从此无效。
  • 也就是说当 func(std::move(a))时，func会匹配到处理右值函数，正常来讲会把a对象内的ptr交付给别处管理，a从此处于无效的，可析构的状态，之后也就用户也就该知道，不可以访问a。否则极有可能异常。
  • 原理
    • 引用折叠 + remove_reference_t + static_cast
    • remove_reference_t获取本来的不带引用的基本类型，然后static_cast为对应右值引用类型。
  • 例子

    forward完美转发 配合 引用折叠使用

  • forward：通过显示模板实参+引用折叠调用，作用：保持（识别）实参的左值/右值信息。
    • 一个【成员函数模板调用forward】可以替代 【一对左值为参数的函数和右值为参数的函数】
  • 原理
    • remove_reference_t
  • forward
  • 例子

引用折叠：

  • move和forward的目的是一致的：正确解析出 左/友值引用所引用的变量究竟是左值还是右值，进而调用相应函数，只不过forward比move更简洁一些。

vector 实现右值push_back

  • vector：push_back右值的原理
    • push_back(&) -> construct(&) -> new T(&)
    • push_back(&&) -> construct(&&) -> new T(&&)（也就是调用 类型的右值拷贝构造，如CMyString(&&))
