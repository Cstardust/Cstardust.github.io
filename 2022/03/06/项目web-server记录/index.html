主要工作

  • 主线程用EPOLL技术实现IO多路复用。监测读socket、写socket事件
  • 处理模式：同步IO模拟Proactor模式
    • 难道不是单reactor多线程
  • 使用有限状态机解析HTTP请求报文，可以解析GET 请求 // 和POST请求。
  • 多线程，增加并发服务数量。

项目难点

  • 实现线程池，提高吞吐量

    • 信号量实现对任务的互斥，互斥锁实现线程同步。

  • 主从状态机对HTTP报文进行解析

    收获

  • 进一步理解TCP三次握手以及四次挥手

  • 对Web服务器服务过程有基本了解

  • 学习了HTTP协议。

  • 提高debug能力，提高C++代码能力，学习Linux系统编程以及网络编程，计网，操作系统知识

  • 理解什么是短连接

    • 长连接短连接

    • 处理完一次client发来的请求并发送完响应之后就立刻断开连接，把负责连接的connection资源释放或者放入对象池。

    • 本例这里将其放回对象池（就是一个先new出来的数组）。当新的（同一）client发来请求后，在server端看起来并没有区别，还是重新造一个conn连接对象。之前处理请求的相关数据都已不在。sever和client看起来就像没见过面一样。
