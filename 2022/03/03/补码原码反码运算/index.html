原码 反码 补码

  • 先说好我的看法

    • 根本就没有所谓的原码和反码，补码也不应该叫做补码，不知道是谁起的名字，还不如叫做真码。(计算机内部真正是以什么样的01序列存储的对应数)

  • 补码：一个数在计算机中真正的编码形式。真正是以什么样的01序列存储的（也就是所谓的补码表示法）。计算机里就是这么表示的，根本没有什么原码和反码。

  • 原码：我们数学人为计算的 一个数的二进制表示，再加上一个符号位。这也就是所谓的原码表示法。

  • 计算机内部为什么将数编码成真码（补码表示法）这样的01序列。（最早最早的时候或许用过原码表示法）

    • 为了使得n位01序列 所能表示的>=0的数和 其所表示的负数个数相同。
    • 为了让0不会有多种编码方式表示。
    • 也就是在原码表示法下，0才有正0和负0，计算机真正使用的编码方式（补码表示法）根本就没有负0

  • 而所谓的原码到反码再到补码的转换方法，不过是为了让我们人类可以快速人工计算出一个数在计算机内部是以什么样的01序列存储的。

  • 原码 -> 补码

    • >=0 原码就是补码
    • <0 补码 = 原码取反 + 1（注意符号位不取反）

  • 补码 -> 原码

    • >=0 原码就是补码
    • <0 原码 = 补码取反 + 1（注意符号位不取反）

  • （移码 = 补码的符号位取反，表示范围和补码一致）

  • 我们来看看所谓的原码表示法（有符号定点整数）

    原码表示法

原码 表示的数
0 1111111 2^7-1
… …
0 0000001 1
0 0000000 0
1 0000000 -0
1 0000001 -1

… …
1 1111111 -(2^7-1)
  • 可以看出 表示原码整数范围 [-2^7-1,2^7-1]。 —> [-2^n-1,2^n-1]
    • 负数：[-2^7-1,-0] —> [-2^n-1,-0]
    • >=0：[0,2^7-1]。 —> [0,2^n-1]
  • 原码小数的范围是 – (1–2^–n) ≤ x ≤ 1–2^–n（n是数值位数）

补码 表示的数
0 1111111 2^7-1
… …
0 0000001 1
0 0000000 0
1 1111111 -1
1 1111110 -2
1 1111101 -3
… …
1 1111111 -(2^7-1)
1 0000000 -2^7
  • 可以看出 范围[-2^7,2^7-1] —> n+1位（数值位为n位）补码表示定点整数范围：[-2^n,2^n-1]
    • 负数：[-2^7,-1] —> [-2^n,-1]
    • >=0 ：[0,2^7-1] —> [0,2^n-1]
  • 多么美好！负数2^8个，>=0的有2^8个
  • 表示的也很自然啊！
    • [00000000,01111111]有2^8个，首位为0，为非负数
    • [10000000,11111111]有2^8个，首位为1，为负数
  • 所以由01111111到10000000时再循环的表示回来 （如表）

同理 n+1位（数值位为n位）补码表示定点小数范围：[-1,1-2^(-n)];

补码作用：用补码把减法变成加法。

  • 原码计算：一个数的原码 - 一个负数的原码 = 一个数的原码 + 这个负数的补码 = 答案的原码。
  • 先说好 计算套路：A - B = A的原码 + （-B）的补码 。其中符号位参与运算，并把结果溢出的位舍去（最高位可能溢出）。得到的结果就是答案的原码。
  • 补码原生定义：模 - a的绝对值 = a的补码
  • 为什么要把减法转化成加法？
    • 因为如果是减法的话机器会变得复杂。
  • 那么如何把减法变成加法？
    • 利用补码。
    • a + ((-b)的原码) = （a - |-b|的原码） = a + （-b）的补码 = 答案的原码
    • a - b的运算 ，将 -b看作一个整体，变成 a + (-b)
      那么-b如何处理？就是把b由原码转化成补码 b = ~b + 1
      之后a + b即可 得到正确结果

纯补码运算—加减

规则

  • 计算时别忘了补码的符号位也占机器字长
  • 符号位参与运算，符号位产生的进位自然丢掉
  • 补码运算：计算时带着符号算，符号位产生的进位自动丢掉。

  • 补码：
    • 正数同原码
    • 负数补码中，
      • 最右边的1以及往右的数同原码 ：即[100000] 同原码
      • 最右边的1往左与原码相反（不包括那个1）：即[xxxxxx1)与原码相反
  • A补 + B补 = ans补，无论正负。
  • [A+B]补 = A补 + B补
  • [A-B]补 = A补 + [-B]补
  • 其真实结果（人类看的）记得要把补码转换为原码

浮点数

  • 阶码和尾数都用原码表示




规格化浮点数

  • 尾数部分：小数点左边为符号位，右边为数值位
  • 解码部分：左侧为符号位，右侧为数值位
  • 规格化：最高数值位为有效值
    • 原码：无论正负，最高位数值位 = 1
    • 补码：最高数值位和符号位相反
      • 正数：最高位数值位 = 1
      • 负数：最高位数值位 = 0
  • 确定真值时，一定要先把补码转化成原码，再求出范围。
    • 规格化0.110 ； 1.1110100
      • 由于尾数用补码表示，所以数值最高位应该和符号位不同，所以算数左移3位，移到1。阶码因此-3.
    • 结果：0.011 ； 1.0100000
  • 阶码用补数表示的情况的阶码范围，就和前面说的补数本身的范围一样。
    • n+1位（数值位为n位）补码表示定点整数范围：[-2^n,2^n -1]
      • 故 阶码最小是 2^(-2^n);
      • 最大时2^(2^n - 1);
    • n+1位（数值位为n位）原码表示定点整数范围：[-2^n-1,2^n-1]

存储器 内存

  • 1.概念问题，内存是存储器的一种类型，可以说是内部存储器。就当前电脑的概念来说，内存是比较直接的让CPU操作的部分（包括CPU里面的一级二级等等缓存）。有个特点，就是通常是带电操作的读写存储器。（寄存器也算是一种，那就更直接的操作了）。而存储器种类就多样了，内存是一种，硬盘，U盘，光盘磁带等等都是的啦。

cpu

  • CPU的工作分为 5 个阶段：取指令阶段、指令译码阶段、执行指令阶段、访存取数和结果写回。
    • 1、取指令（IF，instruction fetch），即将一条指令从主存储器中取到指令寄存器的过程。
    • 2、指令译码阶段（ID，instruction decode），取出指令后，指令译码器按照预定的指令格式，对取回的指令进行拆分和解释，识别区分出不同的指令类 别以及各种获取操作数的方法。
    • 3、执行指令阶段（EX，execute），具体实现指令的功能。CPU的不同部分被连接起来，以执行所需的操作。
    • 4、访存取数阶段（MEM，memory），根据指令需要访问主存、读取操作数，CPU得到操作数在主存中的地址，并从主存中读取该操作数用于运算。
    • 5、结果写回阶段（WB，write back），作为最后一个阶段，结果写回阶段把执行指令阶段的运行结果数据“写回”到某种存储形式。
